HTML Forms

Inputting data with forms

Did you know that there is a <form> tag? The form tag creates a new form for inputting user data to the server.

The <form> tag has an attribute called action. This is the url where the form should be sent. It must also have an attribute called method. This should always be set to method="POST"
Note: this tutorial will not explain how the server processes form data.

In the form content, you must put input tags. These are how the user inputs data.

An input tag should just be a empty tag with an attribute called type. This specifies what kind of input you want the user to enter. Here are some of the important input types:

  • text (default value)
  • password
  • submit
  • reset
  • range
  • radio
  • checkbox
  • file


Type="text" makes a text box where the user can input some data

Type="password" makes a text box, but it will replace the text with dots so that you can't actually see it

Type="radio" makes a multiple-choice input

Type="checkbox" makes a checkbox that the user can check or uncheck.

Type="file" makes a button that will allow the user to input a file.

Type="range" has attributes min and max, which are numbers, and creates a slider that can move from the min to the max values.

Type="submit" creates a button which submits the form to the server at URL in the form's action attribute. Type="submit" can also be used in a <button> element; it does the same thing when inside a form (see lesson 4).

Type="reset" creates a button which resets all other input tags to their default value. Type="reset" can also be used in a <button> element; it does the same thing when inside a form (see lesson 4).

All input tags must have a name attribute and a value attribute. The value attribute will change depending on what the user inputs. When the form is submitted, each input will be sent to the server as name=value, name is the name attribute, and value is the user input or default value.

Inputs can also have a required attribute. This is an empty attribute, it does not need a value. But if it is included, the form will only submit when its input tag is filled out.

Forms can have other tags as well, like paragraph tags to add labels to the inputs. Here is an example: 


<!DOCTYPE html>
<html>
<head>
    <title>HTML Example</title>
</head>
<body>

<form action="/formdata" method="POST">
    
    <p> Text and passwords are easy to make. The name should explain what the input is for, and the value is the default value in the text box: </p>
    
    Username: <input type="text" name="usr" value="username" required><br><br>
    Password: <input type="password" name="pswd" required><br><br>
    
    <p> Radio inputs are more complicated. They must have the same name, but different values. Whichever one is selected's value is sent to the server. The value is not displayed on the page, so you must write it seperately: </p>
    
    <p><strong>Favorite color: </strong></p>
    <input type="radio" name="color" value="red"> Red <br>
    <input type="radio" name="color" value="orange"> Orange <br>
    <input type="radio" name="color" value="yellow"> Yellow<br>
    <input type="radio" name="color" value="green"> Green <br>
    <input type="radio" name="color" value="blue"> Blue <br>
    <input type="radio" name="color" value="purple"> Purple <br><br>
    <button type="submit">Submit</button>
</form>

</body>
</html>
